# Концепція застосунку — "Спільна подорож" (working title)

## Мета

Мобільний застосунок для пари на відстані, який перетворює тижневу подорож
(кожен рухається окремо, різні міста) на спільний досвід: автоматичний
щоденник із фото, ігровий вибір місць, лічильник до зустрічі, фінальний
рекап у форматі Stories.

## Платформа

KMP (Android + iOS), Compose Multiplatform, MVI-архітектура, Navigation 3,
offline-first як наскрізна вимога.

## Контекст використання

Маршрут: Ряшів → Львів → Тернопіль → Вінниця → Жмеринка → Київ → Жмеринка
→ Кишинів. Розтягнутий на тиждень. Обидва переміщуються окремо, мінімум
ручного вводу даних.

## Модулі

### 1. Автентифікація
- Google Sign-In при першому запуску, обов'язковий крок перед усім іншим

### 2. Онбординг (платформо-залежний)
- Показується, якщо дані застосунку порожні
- Дозвіл на доступ до фото
- **Перевірка/інструкція, як увімкнути геолокацію саме для камери**
  (без цього фото не матимуть EXIF-координат, і кластеризація епізодів
  та автощоденник просто не запрацюють) — окремо для Android і iOS,
  бо це системні налаштування камери/фото, не дозвіл самого застосунку
- Пояснення, як налаштувати "Поділитися" з Google Maps в застосунок
  (скріншот/анімація — окремо для Android і для iOS, бо флоу різний)
- iOS окремо: пояснення Share Extension (не завжди одразу видно в списку дій,
  треба показати як додати вручну)
- Дозвіл на нотифікації

### 3. Pairing — об'єднання двох користувачів
- Один створює поїздку → отримує інвайт-код/лінк
- Другий вводить код або переходить по лінку → приєднується
- Ролі: owner (створив) і member
- **Leave Trip** — вийти з поїздки, дані лишаються для другого
- **Delete Trip** — повне видалення, **тільки owner**
- Debug-меню (прихований вхід) — видалення поїздок, очистка кешу,
  форс-онбординг. Тільки для тестів, не потрапляє в реліз

### 4. Автощоденник
- Джерело — EXIF фото (координати + час), без live-трекінгу
- Кластеризація фото в "епізоди" по часу й місцю
- Reverse geocoding (Google Places) → назва місця
- Vision-опис (Gemini чи інша модель із free tier) на основі кількох
  репрезентативних фото епізоду → емоційний абзац "з враженнями",
  не сухий текстовий шаблон
- У кожного свій щоденник, наповнюється повністю автоматично
- **Розблокування дня — симетричне**: день видно обом тільки коли ОБИДВА
  "закрили" свій день (є дані + вони засинкані). Ніхто не бачить чужий
  день раніше свого — чесність важливіша за швидкість

### 5. Матчер місць
- Наповнення пулу — **імпорт через "Поділитися" з Google Maps**
  (share-лінк на місце → застосунок розпізнає й підтягує деталі) +
  ручне додавання
- Тіндер-свайп по одній картці за раз, кожен свайпає незалежно й
  асинхронно — не треба сидіти разом
- **Матч — тільки взаємний.** Обоє свайпнули "так" на ту саму картку →
  місце йде в Match-лист. Обоє "ні" → картка прибирається з колоди
  назавжди. Розійшлись (один так, один ні) → картка лишається в
  колоді й показується знову пізніше — цикл повторюється, поки не
  збіжаться
- Match-лист — не один "фінальний переможець", а список усіх взаємно
  підтверджених місць, накопичується за поїздку

### 6. Свої місця
- Ручне додавання місця/події з нотаткою — та сама "Матчер" колода,
  місця розбиваються по локаціях у списку "Місця"

### 7. Лічильник до зустрічі
- Не "відстань між поточними локаціями", а **днів/км до дня зустрічі**
- Показується як спадна цифра (count-down)
- Фінал — "0 днів → зустрілись"

### 8. Push-нотифікації
- "Ваш спільний день готовий" — шле обом одночасно, коли симетрична
  умова виконалась
- "Залишилось N днів до зустрічі" — на ключові дні
- Без нагадувань про свайпи в матчері — якщо партнер не відповідає,
  це не підганяється пушами, щоб не було нав'язливо

### 9. UI-ритуал відкриття дня
- День заблокований, поки не виконана симетрична умова
- Стан очікування пояснює причину, а не просто крутить спінер
  (партнер ще знімає / дані чекають мережі / генерується текст)
- Розблокування — плавний перехід картка-в-списку → повноекранний вигляд
- Фото проявляються послідовно, одне за одним
- Текст щоденника — ефект друкарської машинки
- Лічильник до зустрічі — count-down анімація наприкінці, як "бонус"

### 10. Фінальний рекап (Stories-формат)
- **Генерується автоматично одразу по завершенню поїздки** (по останньому
  дню маршруту), push-нотифікація "ваш рекап готовий" обом одночасно
- Повноекранні слайди, свайп/тап-навігація як в Instagram Stories,
  прогрес-бари зверху
- Порядок слайдів: інтро з маршрутом → по слайду на кожен день
  (фото-колаж + хайлайт) → "паралельні життя" (порівняння дня side-by-side)
  → переможці матчів → фінал "0 днів, зустрілись"
- Переходи з ефектом глибини (scale+fade, паралакс фото), не просто swipe
- Auto-advance з таймером, можна тапнути раніше або притримати
- Експорт — серія зображень або цілісний файл для збереження/шеру

### 11. Offline-first (наскрізна вимога, не окрема фіча)
- Всі дії (фото-епізоди, свайпи в матчері, свої місця) працюють локально
  без мережі
- Синк відбувається, коли з'являється з'єднання
- Локальний стан — джерело правди, бекенд — тільки синк-шар зверху

## Ключові рішення, вже зафіксовані

- Симетричне розблокування щоденника
- Матчер наповнюється через share-лінк з Google Maps + ручне додавання,
  не через Takeout
- Матч — тільки взаємний (обоє "так"), розбіжність — картка повертається
  в колоду, обоє "ні" — картка прибирається назавжди
- Owner-only delete поїздки, member може тільки Leave
- Лічильник = до зустрічі, не поточна відстань між локаціями
- Vision-based опис фото (не тільки текстовий контекст) для емоційності
- Recap генерується автоматично по завершенню поїздки, без ручного тригера
