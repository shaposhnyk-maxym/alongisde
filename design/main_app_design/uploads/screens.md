# Екрани застосунку

Згруповано по нижньому меню (5 пунктів) + налаштування окремо.
Дизайн-мова — `design-direction.html`: темний ink-канвас, paper-картки
зверху, split-flap акцент для лічильників.

---

## Auth-гейт (одноразовий, послідовний)

### 1. Login
Google Sign-In.

### 2. Onboarding (`feature:onboarding`)
Послідовність кроків (платформо-залежна, деталі — `docs/roadmap.md`
M6/M7): дозвіл на фото → геолокація для камери → share-setup з Google
Maps (+ iOS Share Extension) → дозвіл на нотифікації.

### 3. Create/Join Trip (`feature:pairing`)
Вибір одразу після онбордингу:
- **Create** — тільки дата початку і дата кінця поїздки. Міста НЕ
  вводяться заздалегідь — підтягуються автоматично з геоданих фото під
  час самої поїздки. Після створення — показ інвайт-коду, очікування
  партнера
- **Join** — код вручну, або автозаповнення через deep link/intent,
  якщо перехід стався по посиланню

---

## Нижнє меню (5 пунктів)

### 4. Хоум (`:app`, агрегує дані з усіх фіч)
Зведення: лічильник до зустрічі, поточний день таймлайну, свіжі
матчі. Після завершення поїздки — тут же доступ до Recap.

### 5. Таймлайн (`feature:diary`)
Один екран-карусель замість List+Detail:
`[Лічильник] → [День 1] → [День 2] → ... → [День N]`
- Поточний день — по центру, сусідні дні виглядають збоку, свайп
  перемикає (навігація як по двозв'язному списку)
- Кожна day-картка: колода фото + typewriter-опис + назва міста +
  прапор країни
- Дні, що ще не розблокувались (симетрична умова не виконана) —
  окремий вигляд картки з поясненням стану очікування, а не просто
  замок

### 6. Місця (`feature:places`)
Список, згрупований по локаціях. Наповнюється через "Поділитися" з
Google Maps (система приймає share-лінк → застосунок підтягує деталі
місця й показує картку на підтвердження) + ручне додавання.

### 7. Матчер (`feature:matcher`)
Тіндер-свайп по одній картці за раз, кожен свайпає незалежно й
асинхронно (не обов'язково одночасно). Джерело карток — пул з "Місця".

**Правила переходу картки:**
- Обоє свайпнули "так" на ту саму картку → переходить у Match-лист
- Обоє свайпнули "ні" → картка прибирається з колоди назавжди
- Розійшлись (один так, один ні) → картка лишається в колоді,
  показується знову пізніше — цикл повторюється, поки не збіжаться

### 8. Match-лист (`feature:matcher`, той самий модуль — інший екран)
Список усіх взаємно підтверджених місць. Це не одна "фінальна
картка-переможець", а зростаючий список — може бути кілька взаємних
матчів за поїздку.

---

## Окремо (не в нижньому меню)

### 9. Налаштування (`feature:settings`)
Доступ — іконкою (напр. з Хоум), не окрема вкладка нижнього меню,
щоб не переповнювати панель. Leave Trip (усі) / Delete Trip (тільки
owner — кнопка не показується member'у). Дозволи, профіль.

### 10. Debug Menu
Прихований (5 тапів по версії в Settings), dev-only build.

### 11. Recap Viewer (`feature:recap`)
Stories-формат, автотригериться по завершенню поїздки. Доступ —
через Хоум.

---

## Навігаційна структура (для Navigation 3 графа)

Auth-гейт — послідовний стек, проходиться один раз:
`Login → Onboarding → Create/Join Trip → Хоум`

Головний застосунок — 5 top-level destinations в нижньому меню
(Хоум/Таймлайн/Місця/Матчер/Match-лист), Settings — окремий
destination, відкривається зверху, не частина нижньої панелі.

Recap — не в основній навігації, окремий full-screen destination,
що перекриває все зверху при тригері або за явним переходом з Хоум.

Схема — `navigation-flow.mermaid`.
